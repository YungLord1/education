# Jenkins

Доброго времени суток, уважаемый проверяющий, это моя самая долгая по времени выполнения домашка... Давно я не читал столько документации и не искал столько информации, которой почти нет. Но вроде как справился, правда хотел еще успеть добавить метрики и не успел добавить multibranch(

Итак, что я сделал в этой домашке:
1. Настроил все 3 ВМ для использования jenkins(поставил джаву, докер, подключил воркеров к мастеру, добавил swap space, прошу поправить меня, если он вообще не нужен);
2. Для начала подключил Jenkins к Гитхабу и делал все в другом репозитории(знал, что коммитов будет очень много, не хотел пугать);
3. Написал unit тесты и интеграционные(функциональные) тесты;
4. Написал сам Jenkinsfile, шде описал 6 стейджев: 
    1) lint - проверяет все *.py файлы в папке с помощью линтера flake8 в окружении venv (не успел добавить hadolint для Dockerfile). Соответственно поправил все свои *.py файлы. Наверное все таки можно было обойтись тут без кредов;
    2) Unit tests - Написал юнит тесты и добавил их в Jenkinsfile. Возможно они немного устаревшие по используемым методам, но я нашел подробную статью на хабре и сделал похожие на те, что там были. Также запускаю их в окружении venv;
    3) Build - При каждом запуске собирается Образ docker, который отправляется на docker hub для дальнейшего использования. Для безопасности использовал Credentials;
    4) Deploy - в Jenkinsfile появляется кнопка для деплоя, т.е. настроил мануально, как по требованию в тз. Используется образ, который был создан в Build;
    5) Integration_tests - проверка работоспособности приложения. Если честно, то сначала написал эти тесты, а потом увидел, что надо unit, прошу поправить, если эти вообще не нужны;
    6) Post actions - очистка системы от всего лишнего через docker prune. Также настройка цвета пайплайна, который отправляется в Gitlab.
5. Как уже говорил в идеале хотелос сделать Multibranch pipeline, но не успел. Поэтому стейджи запускал на разных воркерах, думая с точки зрения безопасности и масштабируемости;
6. Подключил Jenkins к Gitlab, устранил пару ошибок с окружениями, настроил запуск пайплайна по коммиту, добавил ручной деплой с тайм-аутом 48 часов;

Большое спасибо за лекцию и полученные знания, а также за будущую проверку, хорошего настроения!:)
P.S. CI/CD - круто!
